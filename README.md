# Cardiac Arrhythmia Classification using Machine Learning

## 📌 Project Overview
This project focuses on classifying **cardiac arrhythmia types** from ECG-derived clinical features using classical machine learning models. The goal is to predict one of 16 arrhythmia classes (including "Normal") based on 279 numerical features extracted from patient ECG recordings.
The project demonstrates a complete pipeline starting from data cleaning and missing-value imputation to outlier handling, exploratory data analysis, dimensionality reduction, and multi-model evaluation.

---

## 📂 Dataset
The dataset used is the **UCI Arrhythmia Data Set**, which contains:
* 452 patient records
* 279 features per record (age, sex, height, weight, QRS duration, P-R/Q-T/P intervals, heart rate, and per-lead ECG waveform measurements across 12 leads)
* 16 target classes:
  * Normal
  * Ischemic changes (CAD)
  * Old Anterior Myocardial Infarction
  * Old Inferior Myocardial Infarction
  * Sinus tachycardia
  * Sinus bradycardia
  * Ventricular Premature Contraction (PVC)
  * Supraventricular Premature Contraction
  * Left bundle branch block
  * Right bundle branch block
  * 1st Degree AV block
  * 2nd Degree AV block
  * 3rd Degree AV block
  * Left ventricular hypertrophy
  * Atrial Fibrillation or Flutter
  * Other

Missing values in the raw data are encoded as `?` rather than left blank.

## 🔄 Workflow

### 1. Data Loading
* Loaded the raw dataset from `arrhythmia.csv`
* Verified dataset shape, dtypes, and structure with `df.info()`

### 2. Data Cleaning
* Replaced `?` placeholders with `NaN`
* Renamed columns to descriptive feature names (Age, Sex, Height, Weight, QRS_Dur, etc.)
* Dropped column 12 (`J` wave) due to a very high proportion of missing values
* Flagged missingness patterns before imputing

### 3. Missing Value Imputation
* Imputed remaining missing values using `SimpleImputer`
* Verified zero remaining nulls after imputation
* Exported the cleaned dataset to `new_arrhythmia.csv`

### 4. Outlier Handling & Data Visualization
* Visualized null-value distribution across columns
* Identified and corrected implausible outliers (e.g. height values of 780 and 608 corrected to 180 and 108)
* Used pairplots, boxplots, and count plots to inspect feature distributions across the 16 classes
* Plotted class distribution as a log-scaled pie chart due to severe class imbalance

### 5. Feature Analysis
* Computed Pearson correlation of each feature with the target class
* Split data into training (80%) and testing (20%) sets
* Standardized features using `StandardScaler`

### 6. Model Building & Training
Trained and compared six classifiers, both on the full feature set and on PCA-reduced features (98% explained variance retained):
* K-Nearest Neighbors
* Logistic Regression
* Decision Tree
* Linear SVC
* Kernelized SVM (sigmoid kernel)
* Random Forest

### 7. Evaluation
* Measured train/test **accuracy** and **weighted recall** for every model
* Generated confusion matrices for each classifier
* Collected all results into a single comparison table
* Visualized train vs. test accuracy for all models in a grouped bar chart

---

## 📊 Results
* Successfully built an end-to-end classification pipeline for a real-world, imbalanced, high-dimensional clinical dataset
* Compared 6 classical ML models across 2 feature spaces (full features vs. PCA-reduced), for 12 total model configurations
* Random Forest and Kernelized SVM generally outperformed simpler baselines, though performance is impacted by significant class imbalance (Normal and CAD dominate the label distribution)
* Exact train/test accuracy and recall scores for each model are available in the `result` table generated in the notebook

## 🚀 How to Run
1. Download `arrhythmia.csv` from the [UCI Arrhythmia Data Set](https://archive.ics.uci.edu/dataset/5/arrhythmia)
2. Update the file path in the data-loading cell to point to your local copy
3. Install dependencies:
   ```bash
   pip install pandas numpy matplotlib seaborn scikit-learn scikit-plot
   ```
4. Run `code.ipynb` top to bottom in Jupyter Notebook or JupyterLab
